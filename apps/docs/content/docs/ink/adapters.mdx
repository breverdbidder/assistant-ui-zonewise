Adapters customize how the runtime persists threads and handles attachments.

## Thread Persistence

For thread persistence across sessions, use [Assistant Cloud](/docs/cloud) or implement a custom `RemoteThreadListAdapter`.

### Assistant Cloud

The recommended approach for apps requiring persistence. Provides cloud-based thread storage with automatic syncing.

```tsx
import { useLocalRuntime } from "@assistant-ui/react-ink";
import { AssistantCloud } from "assistant-cloud";

const assistantCloud = new AssistantCloud({
  baseUrl: process.env.ASSISTANT_CLOUD_URL,
  authToken: () => getAuthToken(),
});

const runtime = useLocalRuntime(chatModel, {
  cloud: assistantCloud,
});
```

See the [Assistant Cloud documentation](/docs/cloud) for setup details.

### In-memory (default)

When no persistence is configured, threads live in memory only — lost when the process exits.

```tsx
const runtime = useLocalRuntime(chatModel);
```

### Custom Persistence

For custom backend storage, use `useRemoteThreadListRuntime` with a `RemoteThreadListAdapter`:

```tsx
import { useRemoteThreadListRuntime } from "@assistant-ui/react-ink";

const runtime = useRemoteThreadListRuntime({
  runtimeHook: () => useMyLocalRuntime(chatModel),
  adapter: myCustomAdapter,
});
```

## Runtime Adapters

Pass adapters to customize runtime behavior:

```tsx
const runtime = useLocalRuntime(chatModel, {
  adapters: {
    history: myHistoryAdapter,
    attachments: myAttachmentAdapter,
  },
});
```

### ThreadHistoryAdapter

Provides message history persistence for individual threads.

```tsx
interface ThreadHistoryAdapter {
  load(): Promise<ExportedMessageRepository>;
  append(item: ExportedMessageRepositoryItem): Promise<void>;
}
```

## Which option to choose?

| | `useLocalRuntime` (in-memory) | `useLocalRuntime` + Cloud | `useRemoteThreadListRuntime` |
|---|---|---|---|
| **Thread storage** | Memory only | Assistant Cloud | Your backend |
| **Cross-session persistence** | No | Yes | Yes |
| **Setup complexity** | Minimal | Low | Moderate |
| **Best for** | CLI tools, demos | Apps with user accounts | Custom backends |
