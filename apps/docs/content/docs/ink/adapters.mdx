Adapters customize runtime behavior. They can be passed as options to `useLocalRuntime` or `useRemoteThreadListRuntime`.

## RemoteThreadListAdapter

Title generation is configured via the `generateTitle` method on `RemoteThreadListAdapter`. See the [Custom Backend](/docs/ink/custom-backend) page for a full example.

```tsx
import type { RemoteThreadListAdapter } from "@assistant-ui/react-ink";
import { createAssistantStream } from "assistant-stream";

const myAdapter: RemoteThreadListAdapter = {
  // ... other methods ...

  async generateTitle(remoteId, unstable_messages) {
    return createAssistantStream(async (controller) => {
      const res = await fetch(`/api/threads/${remoteId}/title`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ messages: unstable_messages }),
      });
      const { title } = await res.json();
      controller.appendText(title);
    });
  },
};
```

## FileStorageAdapter

For terminal apps that need local thread persistence without a backend, use `createFileStorageAdapter`. Thread metadata is stored as JSON files on disk — threads survive process restarts.

```tsx title="app.tsx"
import {
  useLocalRuntime,
  useRemoteThreadListRuntime,
  AssistantRuntimeProvider,
  createFileStorageAdapter,
} from "@assistant-ui/react-ink";
import { myChatAdapter } from "./adapters/my-chat-adapter.js";

const fileAdapter = createFileStorageAdapter({
  dir: "./data/threads",
  prefix: "@assistant-ui:",
});

function useAppRuntime() {
  return useRemoteThreadListRuntime({
    runtimeHook: () => useLocalRuntime(myChatAdapter),
    adapter: fileAdapter,
  });
}

export function App() {
  const runtime = useAppRuntime();
  return (
    <AssistantRuntimeProvider runtime={runtime}>
      {/* your chat UI */}
    </AssistantRuntimeProvider>
  );
}
```

### Options

| Option | Type | Description |
|--------|------|-------------|
| `dir` | `string` | Directory for thread files |
| `prefix` | `string \| undefined` | Prefix for storage keys |

### Using FileStorage directly

For more control, use the `FileStorage` class directly with `createLocalStorageAdapter`:

```tsx
import { createLocalStorageAdapter } from "@assistant-ui/core/react";
import { FileStorage } from "@assistant-ui/react-ink";

const storage = new FileStorage({ dir: "./data/threads" });

const adapter = createLocalStorageAdapter({
  storage,
  prefix: "@my-app:",
});
```

Useful for customizing storage behavior or integrating with existing infrastructure.

## Which option to choose?

| | ChatModelAdapter + `useLocalRuntime` | FileStorageAdapter | RemoteThreadListAdapter |
|---|---|---|---|
| **Thread storage** | In-memory | Local filesystem | Your backend |
| **Message storage** | In-memory | In-memory | In-memory (can add history adapter) |
| **Cross-session persistence** | No | Yes | Yes |
| **Setup complexity** | Minimal | Minimal | Moderate |
| **Best for** | Demos, prototypes | CLI tools, single-user apps | Production apps, team sharing |
